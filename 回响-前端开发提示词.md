## 全局设计规范（所有页面共用，每个页面提示词开头粘贴此段）

```
微信小程序界面。暖白背景 #FAF9F6。文字主色 #4A4540 深棕灰。
辅助文字 #9E9A96 中灰。弱化文字 #C5C0BB 浅灰。
点缀色 #C87A5A 暖橙——仅用于重要按钮和信纸装饰。
卡片白色 #FFFFFF，阴影 rgba(0,0,0,0.03)。圆角16px。
系统默认无衬线字体（PingFang SC）。
页面标题13px，每日问题20px/行高1.6/字重500。
用户输入16px/行高1.8/字重400。
核心一句话22px/行高1.4/字重600。
完整回信正文16px/行高2.0/字重400。
落款13px/字重400。
触控目标≥44px。大量留白。页面内边距20-28px。
无多余动效。无弹窗。无自动播放。无打卡提醒。
只有信纸到达时有轻柔动画。
```

## 页面一：首页（每日一问 + 写作区）

```
微信小程序界面。375×812px。
使用全局设计规范。

【页面结构——从上到下】

1. 顶部状态栏区域留白，无传统导航栏。页面左上角两个小字"回响"，字号13px，颜色 #C5C0BB，字间距4px。页面右上角一个小信封图标（24px），点击进入信箱。信封图标颜色 #9E9A96。

2. 日期行。居中显示"6月11日 · 星期三"，字号13px，颜色 #C5C0BB。距离顶部约60px。

3. 问题卡片。日期下方约32px。白色卡片，圆角20px，阴影 rgba(0,0,0,0.03)，内边距28px。
卡片内容：一个小巧的emoji或圆点装饰（位于卡片顶部居中，字号20px），然后是今日问题——
"上一次你感到真正的快乐，是在做什么？" 字号20px，行高1.6，字重500，颜色 #4A4540，居中。
问题下方留白充足（约16px）。

4. 写作区域。问题卡片下方间距20px。一个白色卡片，圆角16px，边框颜色 #EBE8E4，边框宽度1px，内边距20px。
卡片内部是一个 textarea 输入框，宽100%，无边框，无轮廓，背景透明，字体大小16px，行高1.8。
placeholder 文字"写下你的回答…" 颜色 #C5C0BB。textarea 最小高度120px。
没有字数统计。没有格式工具栏。没有保存草稿按钮。
如果用户清空输入框或什么都不写直接投递，系统自动处理（发送一封"沉默的信"）。

5. 投递按钮。写作区下方间距24px。
一个暖棕色 #4A4540 的圆角按钮，文字"投进信箱"，白色字，字号16px，字间距1px，居中。
按钮宽度为页面宽度的60%，height 48px，圆角14px。
按钮下方小字："回信需要一点时间" 字号11px，颜色 #C5C0BB，居中。
点击按钮后 → 进入等待页。

6. 底部没有标签栏。干净收尾。

【交互状态】
- 问题卡片加载时：先显示卡片骨架，问题文字淡入（opacity 0→1，持续0.5s）
- 输入框聚焦时：边框颜色从 #EBE8E4 变为 #C87A5A
- 按钮点击时：轻微缩放（scale 0.97），颜色不变
- 按钮禁用态：当用户未输入任何内容时按钮仍可点击（允许空投递）
- 页面滚动：整页可滚，没有固定元素（除顶部外）

【数据加载】
- 页面 onLoad 时从云数据库获取今日问题
- 如果用户今天已经写过：直接显示今天的回信页，不显示问题卡片和写作区
```

## 页面二：等待页（投递后的过渡动画）

```
微信小程序界面。375×812px。全屏。
使用全局设计规范。背景色 #FAF9F6 铺满整个屏幕。

【页面结构】

1. 整个页面垂直居中排列。无导航栏。无按钮。纯展示。

2. 信封图标（48px，使用 emoji ✉️ 或 SVG 信封图形）位于屏幕中央偏上。
图标带有轻柔浮动动画：
  animation: float 3s ease-in-out infinite;
  @keyframes float { 0%,100% { transform: translateY(0); } 50% { transform: translateY(-12px); } }
颜色 #C87A5A。

3. 信封下方间距约40px。一首三行诗逐行显示。
每行间隔约1.5秒出现，从 transparent 淡入到 #4A4540。
字体大小16px，行高2.0，居中。

三行诗示例（从预置诗库中随机抽取一首）：
"把今天的话
 折成纸船
 明天漂回来"

三行全部显示完毕后，停留约1.5秒。

4. 然后整个页面以淡出动画（opacity 1→0，持续0.6s）过渡到回信页。

【动画参数】
- 信封浮动：循环3s，ease-in-out
- 每行诗淡入：opacity 0→1，持续0.8s，easing: ease-out
- 页面过渡：opacity 1→0，持续0.6s，然后 navigateTo 回信页

【实现要点】
- 此页面没有任何用户交互。纯动画页面。
- 在此页面展示期间，后台已发起 DeepSeek API 请求。
- 动画总时长（1.5s×3 + 1.5s + 0.6s ≈ 6.6s）足够API返回结果。
- 如果API提前返回，动画完成后直接展示结果。如果API超时（>10s），动画结束后显示"信还在路上…"并加一个加载指示器。
```

## 页面三：回信页（信的内容展示）

```
微信小程序界面。375×812px。滚动页面。
使用全局设计规范。

【页面结构——从上到下】

1. 顶部无导航栏。左上角一个小返回箭头（←），颜色 #9E9A96，点击回到首页。字号16px。

2. 信纸容器。整个回信放在一个白色大卡片内。圆角20px，阴影 rgba(0,0,0,0.05)（比首页卡片稍深以体现信的厚实感）。内边距32px。
背景色 #FFFEFC（比纯白暖一点点，模拟信纸色）。
信纸底部有一条细微的折痕效果：一个浅灰横线 rgba(0,0,0,0.04) 位于距离底部约40px处。

3. 信纸内部从上到下：
   a. 日期行：右上角小字"6月11日"，字号12px，颜色 #C5C0BB。
   b. 核心一句话：日期下方约20px。字号22px，行高1.4，字重600，颜色 #3A3530。居中。这是回信的灵魂——一句话。
      示例："那一晚你不是在打游戏。你是在想你朋友了。"
   c. 分隔线：核心一句话下方约16px。一条细线，颜色 rgba(0,0,0,0.06)，宽度60%，居中。
   d. 完整回信：分隔线下方约16px。字号16px，行高2.0，字重400，颜色 #4A4540。
      段落之间有自然留白（每个段落之间空一行）。
      示例：
      "我看到你写的了。你说上一次真正的快乐是跟朋友打了一整晚游戏。那一晚你不是在赢游戏，你是在找一个理由待在朋友身边。你比自己想象中更需要被陪伴。"
   e. 落款：完整回信下方约24px。右对齐。字号13px，颜色 #C5C0BB。
      "—— 来自明天的你"

4. 信纸下方间距约20px。
   两个按钮横排，间距12px：
   a. 左侧——"📥 收进信箱"。浅灰背景 #F0EEEA，文字颜色 #555。圆角14px，height 44px。flex:1。
   b. 右侧——"📤 分享这句话"。暖棕背景 #4A4540，白色文字。圆角14px，height 44px。flex:1。
   
   点击"收进信箱"：保存回信到邮箱，按钮变为"✓ 已收进信箱"（绿色 #34C759），1.5s后恢复。
   点击"分享这句话"：生成一张分享卡片图片（核心一句话+信纸背景+日期+回响Logo），调用微信分享接口。

5. 如果是免费用户——完整回信部分模糊处理（高斯模糊 blur 4px），下方显示一个解锁提示卡片：
   "解锁完整回信" + "¥268/年 · 每天不到8毛钱" + "立即升级"按钮（暖橙色 #C87A5A）
   核心一句话始终可见（这是免费版的价值）。

【信纸背景变化】
- 每封回信随机使用一张信纸纹理（12张中随机）。
- 信纸纹理是通过CSS实现的细微差异：不同的细微纹理图案、不同的纸边不规则度、不同的泛黄程度。
- 节日当天自动使用节日限定信纸。
- 信纸类型记录在回信数据中，用户可在信箱中看到每封信的信纸。

【交互状态】
- 页面入场动画：信纸从 opacity:0, translateY(20px) → opacity:1, translateY(0), 持续0.6s
- 按钮点击：scale(0.97)
```

## 页面四：信箱（历史回信列表）

```
微信小程序界面。375×812px。滚动列表页面。
使用全局设计规范。

【页面结构】

1. 顶部导航栏：白色背景（#FFFFFF），顶部居中标题"信箱"字号17px字重600颜色 #4A4540。
   左侧返回箭头←（点击回到首页）。

2. 导航栏下方是回信列表。每封信一个卡片，纵向排列，间距8px，左右内边距16px。

3. 每封信的卡片：
   白色背景 #FFFFFF。圆角16px。阴影 rgba(0,0,0,0.02)。内边距16px。
   卡片内部从左到右排列：
   a. 左侧：一个小的信封emoji图标（20px），颜色根据信纸类型变化。如果是"本周的回声"标记的信，信封图标替换为一个小星星 ✨。
   b. 中间（flex:1）：上方是问题摘要文字（比如"上一次你感到真正的快乐…"），字号14px，颜色 #4A4540，单行截断。下方是日期"6月11日"，字号12px，颜色 #9E9A96。
   c. 右侧：核心一句话的简短预览（比如"你不是在赢游戏…"），字号12px，颜色 #9E9A96，最多显示15个字，超出省略。右对齐。

   点击卡片 → 展开该封信的完整回信（navigateTo 回信详情页，复用回信页组件）。

4. 释怀标记：
   对于用户标记为"已释怀"的信，卡片整体 opacity:0.5，卡片左侧有一个小绿勾 ✓。
   标记操作：长按卡片 → 弹出底部菜单"标记为已释怀"。

5. 搜索栏（信箱顶部、标题下方）：
   一个浅灰色圆角输入框，背景 #F0EEEA，圆角10px，高度36px。
   左侧放大镜图标。placeholder "搜索回信…"。
   输入关键词后实时过滤列表。

6. 空状态：如果信箱里没有信，显示居中的空状态——
   一个开着的空信封图标（48px，灰色 #D0CCC6）。
   下方文字"信箱还是空的"，字号14px，颜色 #9E9A96。
   再下方"每天回答一个问题，这里会越来越满"，字号12px，颜色 #C5C0BB。

【数据加载】
- onLoad 从云数据库获取当前用户的所有回信记录
- 按日期倒序排列
- 支持下拉刷新（onPullDownRefresh）
- 支持上拉加载更多（如果超过20条）
```

## 页面七：空状态 & 通用组件

```
微信小程序界面。375×812px。
使用全局设计规范。

=== 组件A：通用空状态 ===
居中排列。从上到下：
1. 图标：使用对应场景的图标（信箱空=📭 48px，网络错误=🌐 48px，无数据=📋 48px）
2. 主文字：字号14px，颜色 #9E9A96，居中。"信箱还是空的"
3. 副文字：字号12px，颜色 #C5C0BB，居中。"每天回答一个问题，这里会越来越满"
4. （可选）操作按钮：暖棕背景 #4A4540 的圆角按钮，白色文字。只在需要引导操作时显示。

=== 组件B：通用Toast ===
固定在屏幕底部居中。距离底部约80px。
深色半透明背景 rgba(60,60,67,0.92)，白色文字，字号14px。
圆角20px。内边距10px 20px。自动2秒后消失。
动画：从 translateY(120px) → translateY(0)，持续0.35s，ease-out。

=== 组件C：加载骨架屏 ===
```
