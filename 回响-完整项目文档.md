# 回响 — 完整项目文档 v3.1

## 名字

**回响。**

你对自己说的话，隔天变成一封信，回到你手里。

不是AI在写。不是机器在回。是你的声音穿过时间，绕了一圈，回到你耳边。

---

## 一、项目概述

**产品名：** 回响

**一句话：** 每天一个问题。来自明天的回信。

**产品形态：** 微信小程序

**核心体验：** 每天晚上，你收到一个问题。你回答。几分钟后，你收到一封信——开头写着"亲爱的现在的我"，落款写着"来自明天的你"。

**跟所有其他情感App的根本区别：**

| 别人 | 回响 |
|------|------|
| 你跟AI聊天 | 你跟明天的自己对话 |
| 机器即时回复 | 一封信在等你 |
| 聊完就没了 | 每封信收进信箱，变成你的情绪地图 |
| 你不知道该说什么 | 每天一个问题，帮你开始 |
| 没人能分享 | 回信里最触动的那句话，一键变成分享卡片 |

**目标用户：** 18-30岁年轻人。有自我意识、有情绪深度、不想发朋友圈但想被理解。性别分布预计女性65%+。

**解决的核心问题：** 现代年轻人的情绪没有出口。朋友圈是表演场，微信是工作台，抖音是时间黑洞。没有一个安静的地方——让她们被问到一个真正值得想的问题，然后被认真回应。

## 三、创新点

### 创新一：不是AI回信，是"明天的你"回信

| 所有AI情感产品 | 回响 |
|-------------|------|
| "我是你的AI助手" | "我是明天的你" |
| 第三人称、机器感 | 第一人称、自己跟自己对话 |
| 用户评判信写得好不好 | 用户觉得"这好像真的是我会说的话" |

**信的开头：** "亲爱的现在的我，"
**信的结尾：** "—— 来自明天的你"

**为什么这个创新重要：** 它解决了根本的信任问题。用户不相信AI。但用户愿意相信"一个更了解自己的自己"。AI只是一个翻译器——把用户写的东西翻译成"另一个自己"会说的话。

### 创新二：信纸收藏系统

每封回信随机使用一张信纸。信纸模拟真实纸张的纹理和笔触。

| 信纸类型 | 来源 |
|---------|------|
| 12张基础信纸 | 每月更新一张 |
| 节日限定信纸 | 春节/情人节/清明/端午/中秋/圣诞/元旦/生日——自动匹配当天 |
| 城市信纸 | 用户手动选择所在城市——雨天/晴天/雪天匹配不同信纸 |
| 隐藏信纸 | 连续7天/30天/100天/365天各解锁一张 |

用户收集了多少张信纸显示在"我的"页面。这是唯一的游戏化元素。不强调、不推送"还差X张"。安静地放在那里，让用户自己发现。

### 创新三：一句话就够了

每封回信的核心只有一句话。放在最上方，最大字号。像一个电影台词。

用户不需要读完整封信。看到那一句话就够了。展开的段落是可选的。

分享的也是"这句话"——用户分享的不是App广告，而是一句触动自己的话。看到的人只会问"这是什么"。

### 创新四：每周回声

每周日晚上，推送一条特殊消息：

> "这周你写了X封信。最触动你的是哪一封？"

用户可回顾本周所有回信，选择一封标记为"本周的回声"——这封信会高亮显示在信箱里。

### 创新五：晚上九点的仪式感

不是早上——早上是压力的开始。晚上九点是一天的句号。用户收到推送，推送内容就是问题本身。不需要打开App就能看到问题。如果被触动了，ta会打开。

## 四、完整功能清单

### 4.1 每日一问

| 功能 | 说明 |
|------|------|
| 推送方式 | 付费用户：每晚9点微信服务通知。免费用户：自己打开 |
| 推送内容 | 通知栏直接显示问题本身。用户不需要打开App就能看到 |
| 问题库 | 365个原创问题。按自我认知/人际关系/成长焦虑/未来迷茫/情绪管理五类轮换 |
| 问题深度 | AI根据用户连续使用天数自动调节——第1天浅、第7天中、第30天深 |
| 示例问题 | "你今天最想逃避的一件事是什么？""上一次你为自己骄傲，是什么时候？""你现在最怕谁对你失望？" |
| 可以不回答吗 | 可以。不打卡、不追责。但如果你不写，回信会更短，开头会是"你今天不想说话。没关系。" |

### 4.2 写信（用户输入）

| 功能 | 说明 |
|------|------|
| 输入方式 | 纯文本。不限字数。支持换行。无格式栏 |
| 设计风格 | 一张白纸。没有字数统计。没有保存草稿按钮 |
| 可以空着吗 | 可以。点"投递"——系统自动生成一封"沉默的信" |

### 4.3 回信

| 功能 | 说明 |
|------|------|
| 等待时间 | 约5秒。不显示倒计时。显示一首逐行出现的三行诗 |
| 回信结构 | ①核心一句话（22px大字号）②完整回信（17px，100-200字）③信纸背景④落款"来自明天的你" |
| 回信调性 | "我看到你写了XX"开头。不说"根据分析"、不说"建议你"。像一个了解你的人写给你的话 |
| 免费版回信 | 只展示核心一句话（约30字）。想读完整回信→付费 |
| 付费版回信 | 完整回信 + 信纸收藏 + 可导出 |

### 4.4 信箱

| 功能 | 说明 |
|------|------|
| 排列 | 按日期倒序。每周日的那封有特殊标记（"本周的回声"） |
| 每封信展示 | 日期 + 问题摘要 + 核心一句话（最醒目） |
| 点击展开 | 完整回信 + 用户当时的回答 + 信纸 |
| 释怀标记 | 对于"曾经担心"的信，可标记"已释怀"。信还在，视觉变淡 |
| 免费版限制 | 信箱只保留最近7封。第8天第一封信消失 |
| 导出 | 付费用户可导出全部回信为PDF（v2功能，开发中） |

### 4.5 每周回声

| 功能 | 说明 |
|------|------|
| 触发 | 每周日晚上 |
| 推送内容 | "这周你写了X封信。最触动你的是哪一封？" |
| 交互 | 用户可回顾本周所有回信。可选择一封标记为"本周的回声" |

### 4.6 信纸收藏

| 功能 | 说明 |
|------|------|
| 基础信纸 | 12张。每月新增1张 |
| 节日限定 | 春节/情人节/清明/端午/中秋/圣诞/元旦/用户生日——当日自动匹配 |
| 城市信纸 | 用户手动选城市。不选则默认 |
| 隐藏信纸 | 连续7天/30天/100天/365天各解锁一张 |
| 查看 | "我的"页面→信纸收藏。不推送"还差X张" |

### 4.7 分享

| 功能 | 说明 |
|------|------|
| 分享内容 | 不是分享App。不是分享用户的回答。是分享回信里的核心一句话 |
| 分享形式 | 一张精美卡片——那句话 + 信纸背景 + 日期 + 回响Logo |
| 看到的人会怎样 | 看到一句话。如果被触动，会问"这是什么"。然后打开小程序 |

### 4.8 我的页面

| 功能 | 说明 |
|------|------|
| 统计 | 写了多少封信 + 连续使用了多少天 + 收集了多少张信纸 |
| 信纸收藏 | 已收集/总数量。点击查看全部 |
| 功能入口 | 导出信箱（即将上线）+ 推送设置 + 城市设置 + 订阅管理 |
| 订阅信息 | 当前版本 + 升级入口 |

### 4.9 订阅系统

| 版本 | 价格 | 内容 |
|------|------|------|
| 免费版 | ¥0 | 每天可用。回信只显示核心一句话（约30字）。信箱保留7封。基础信纸3张 |
| 专业版（月度） | ¥39/月 | 完整回信。信箱永久保存。全部基础信纸12张。节日限定信纸。每周回声 |

**定价心理学：**

| 策略 | 操作 |
|------|------|
| 锚定 | 用户先看到¥39/月。¥268/年=每月¥22.3——立刻觉得年度版便宜了43% |
| 对比 | 年度版旁边标注"比按月支付省¥200"（39×12-268=200） |
| 推荐 | 年度版标注"最受欢迎"标签 |
| 话术 | 核心展示不是"¥268/年"，而是"每天¥0.73" |
| 免费版钩子 | 能收到一句话，但完整回信就在那里。用户知道"只差一步" |
| 7天信箱清空 | 免费版信箱只保留7天。用户写到第6天，信箱里有6封信——第8天第一封信会消失。ta不会想失去它们 |

**购买页视觉顺序：**
1. 年度版 ¥268/年 最上方（大卡片 + "最受欢迎"标签 + "每天¥0.73"）
2. 月度版 ¥39/月 下方（小卡片，无标签）

## 五、UI设计思路

### 5.1 整体调性

| 关键词 | 体现 |
|--------|------|
| 安静 | 无多余动效、无弹窗、无自动播放、无打卡提醒 |
| 温暖 | 暖白底 + 棕灰文字 + 暖橙点缀 |
| 纸的感觉 | 信纸纹理（CSS模拟）、轻微阴影模拟纸张厚度 |
| 高级 | 字间距、行间距精心调试。像一本排版过的书 |
| 不打扰 | 每天最多一条推送。没有"你已连续X天"的压力 |

### 5.2 色彩系统

| 用途 | 色值 |
|------|------|
| 页面背景 | #FAF9F6 暖白 |
| 文字主色 | #4A4540 深棕灰 |
| 辅助文字 | #9E9A96 中灰 |
| 弱化文字 | #C5C0BB 浅灰 |
| 核心一句话 | #3A3530 最深的棕 |
| 点缀色 | #C87A5A 暖橙——只用于重要按钮和信纸装饰 |
| 卡片 | #FFFFFF 白，阴影 rgba(0,0,0,0.03) |

### 5.3 字体和排版

| 元素 | 字号 | 行高 | 字重 |
|------|------|------|------|
| 页面标题 | 13px | — | 400 |
| 每日问题 | 20px | 1.6 | 500 |
| 用户输入 | 16px | 1.8 | 400 |
| 核心一句话 | 22px | 1.4 | 600 |
| 完整回信 | 16px | 2.0 | 400 |
| 落款 | 13px | — | 400 |
| 辅助文字 | 13px | — | 400 |

### 5.4 页面结构

| 页面 | 内容 |
|------|------|
| 首页 | 顶部"回响"两个小字 + 日期。中间问题卡片。下方写作区（一张白纸）。底部"投递"按钮。无传统导航栏 |
| 等待页 | 一首诗逐行出现。信封图标轻柔浮动 |
| 回信页 | 核心一句话（最大字号）→ 完整回信 → 信纸背景 → 落款"来自明天的你" → 保存/分享按钮 |
| 信箱页 | 导航栏"信箱"。回信列表按日期倒序 |
| 我的页 | 导航栏"我的"。统计数据 + 信纸收藏 + 功能菜单 + 订阅管理 |

### 5.5 动画

| 场景 | 动画 |
|------|------|
| 投递信封 | 信封图标轻微弹跳→缩小→消失 |
| 等待中 | 一首诗缓缓浮现，逐行显示。信封图标在背景轻柔浮动 |
| 回信到达 | 信封淡入→缓缓展开→信纸呈现 |
| 翻页 | 信箱中的信点击后从右侧滑入 |
| 释怀 | 信件颜色变淡，一个轻微的沉入动画 |

## 六、技术方案

### 6.1 架构

```
用户 → 微信小程序 → 微信云函数 → DeepSeek API
                        ↑
                   API Key在这里
                   用户永远看不到
```

| 组件 | 方案 | 说明 |
|------|------|------|
| 前端 | 微信小程序原生开发 | WXML + WXSS + JS |
| AI引擎 | DeepSeek API | V3.2-Exp模型 |
| 后端 | 微信云开发（云函数） | 免费额度内够用 |
| 数据库 | 微信云数据库 | 存用户数据、回信记录、信纸信息 |
| 推送 | 微信服务通知 | 付费用户每日推送 |
| 支付 | 微信支付 | 对接订阅制 |
| 信纸渲染 | CSS + Canvas | 小程序端生成 |
| PDF导出 | 小程序Canvas | v2功能 |

### 6.2 API Key管理

1. 开发者在 platform.deepseek.com 注册并获取API Key
2. Key配置在微信云函数环境变量中
3. 小程序前端调用云函数 → 云函数携带Key请求DeepSeek
4. 用户端完全不知道Key的存在

| 冷启动 | 100人 | 50人 | ≈¥55 |
